**Problem**: Current logic uses `strings.ReplaceAll`, `TrimSuffix`, and manual filename appending. Fragile with edge cases (spaces, symlinks, trailing slashes, Windows paths in tests).

**Action**:
- Replace string manipulation with `filepath.Join`, `filepath.Rel`, and proper tar header normalization
- Add test cases for paths with spaces, dots, and nested directories
- Consider using `tar` package's built-in path cleaning where possible
